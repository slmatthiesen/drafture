# Trade-monitoring handoff pack (Drafture dogfood)

This is the **agent-handoff pack** proof-of-concept, built from a real Drafture
generation (`qg7zz9UuLpPL` — TradingView trade ingest + AI eval, budget tier).

## What's in here

| File | What it is |
|---|---|
| `DRAFTURE.md` | **Agent brief — read first.** The architecture, the intentional scope decisions (incl. the dashboard read-API gap), the security floor, and the gaps the agent must close. |
| `budget.tf` | Reference-only Terraform for the budget tier, generated by GLM (free). |
| `design.json` | The full Drafture design body (all 3 tiers, keyDecisions, security floor) — machine-readable reference. |

## The thesis being tested

Drafture ships **judgment, not just diagrams.** For that judgment to survive into the
build, the agent that finishes the work needs Drafture's *decisions and scope boundaries*
— not only the raw `.tf`. Otherwise the same scoping bug propagates: an agent handed only
`budget.tf` can't tell that the missing dashboard read-path is an **open gap to raise**,
not a feature to skip or silently reinvent.

`DRAFTURE.md §2` is the load-bearing piece: it tells the agent what's deliberately out of
scope vs. what's an unresolved gap to flag.

## How to run the dogfood

1. Point a coding agent (Claude Code) at this directory.
2. Prompt: *"Read DRAFTURE.md first, then review budget.tf against it. Tell me what you'd
   change to make it plan-ready, and surface anything DRAFTURE.md flags as an open gap."*
3. The pass/fail signal: does the agent **raise the dashboard read-API gap on its own**
   (citing §2), and does it **stop at `terraform plan`** rather than auto-applying? If yes,
   the handoff thesis holds.

## Regenerating the `.tf`

`budget.tf` was generated via the GLM provider (`LLM_PROVIDER=glm`, `LLM_MODEL=glm-4.5-flash`,
~38s, $0). To regenerate any tier, open that tier's config panel in the Drafture UI — it
generates + caches the `.tf` on the generation row (the throwaway generator script used
here was deleted; the production path is the `/api/config` route).
